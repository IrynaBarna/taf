Docstring

All files, classes, class methods and first level functions must have properly created docstrings. Note that 'type' syntax in Python docstrings is not defined by any standard. Thus, suggest following notations at https://www.jetbrains.com/pycharm/webhelp/type-hinting-in-pycharm.html, http://blog.jetbrains.com/pycharm/2013/02/dynamic-runtime-type-inference-in-pycharm-2-7/, http://epydoc.sourceforge.net/fields.html. This type hinting is only for TAF core library, not for testcases.

File

Each python file in TAF should contain a header where the main information about the file is stored. Following keywords can be used:

  • @copyright - put Intel copyright statement;
  • @file - the name of the test suite (python file name, e.g.: @file <test_suite_name>.py);
  • @summary - the summary of the test suite;
  • @details - list available test cases in the test suite.

3

19

23  2. Verify that static VLAN can be deleted and default VLAN cannot be deleted
24 
25 """

Class

Create a class with appropriate docstring. Following keywords can be used:

  • @description - describing the functionality of testing suite;
  • @par - paragraph for further detailed explanation or to give examples;
  • @code, @endcode - to mention any coding samples for the usage.

Example:

1 """
2 @description General Switch object functionality.
3 
4 @par Configuration examples:
5 
6 @code{.json}
7 {
8 "name": "simswitch2_lxc",
9 "entry_type": "switch",
10 "instance_type": "lxc",
11 }
12 @ endcode
13 """

Function

Test Case

Create a test case function with appropriate docstring. Following keywords can be used:

  • @brief - summary of the particular test case should explain actual device's behavior;
  • @steps - describe the steps for particular testcase.

Example #1 (TAF Core library)

1 def cli_set(self, commands, split_lines=True, expected_rc=0)
2  """
3  @brief: Sends a list of commands, will raise an exception on any error.
4  @param commands: command string to execute
5  @type commands: list[list[str]]
6  @param split_lines: determine to split or not.
7  @type split_lines: bool
8  @param expected_rc: expected return code
9  @type expected_rc: int
10  @raise UIException
11  @rtype: list[list[str]]
12  @return: Output in sequence per list of commands
13  """

Example #2 (Test Case)

1 """
2 @brief Verify that the Static entry can be removed
3 @steps
4  -# Disable STP.
5  -# Add new static entry to FDB.
6 @endsteps
7 """

Sub module Functions

Sub-functions inside first level functions need not contain doc strings as far as they aren't designed for any external calls. Ignore pylint messages. In case you wish to create a docstring, following keywords can be used:

  • @brief - summary of the particular function;
  • @param - explanation of arguments given to a function;
  • @type - explanation of argument type given to a function;
  • @note - any notes for better understanding;
  • @code, @endcode - to mention any coding samples for the usage
  • @rtype - to specify return type of the function
  • @return - to specify return value of the function

Example:

1 """
2 @brief Check that FDB table is filled correctly
3 @param switch_instance Switch instance to work with
4 @param macaddress MAC address for check
5 @note This function check if master port should be devided into slave ports.
6 @return Count of entries
7 @code
8 assert self._is_entry_added_to_fdb_table(portid=ports[('sw1', 'tg1')][1], macaddress=source_mac, vlanid=vlan_id, fdb_type="Dynamic", switch_instance=env.switch[1]) == 1
9 @ endcode
10 """
  • Use @copydoc <link-object> command to avoid cases where a documentation block would otherwise have to be duplicated or to extend the documentation of an inherited member.
  • In order to copy the documentation for a member of a class:
1 def myfunction():
2  """
3  @copydoc MyClass::myfunction()
4  More documentation if required
5  """
  • In case if source docstring is in other file, you can use the following syntax:
1 def customized_get_file():
2  """
3  @copydoc testlib::cli_template::CLIGeneric::get_file()
4  More documentation if required
5  """

Where testlib is file's folder, cli_template is file name, CLIGeneric is class, get_file() is function.
